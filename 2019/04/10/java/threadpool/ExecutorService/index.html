ExecutorService

ExecutorService是对线程池定义的接口。接口定义了后台任务执行的相关方法。

方法说明

shutdown()

线程池使用完之后，应该进行关闭。调用shutdown方法，ExecutorService不会马上关闭，但是不再接收新的任务，直到当前线程执行完成之后才会关闭。
所以shutdown之前的提交的任务都会被执行。

shutdownNow()

执行这个方法的时候，会跳过正在执行的任务和被提交还没有执行的任务，然后依次执行线程的interrupt中断方法。但是只会中断阻塞的方法，对正在执行的任务，
并不能保证是否能停止。并且会返回未执行的任务。

isShutdown()

当调用shutdown或shutdownNow方法后返回为true。

isTerminated()

当调用shutdown后，并且所有提交的任务都完成后返回true。
当调用shutdownNow后，成功停止后返回true。

awaitTermination(long timeout, TimeUnit unit)

等待设置的时间，来判断当前任务是否执行完。一般与shutdown搭配使用来判断是否任务执行完成。

execute(Runnable command)

异步执行任务，但是没有办法获取任务执行的结果。

submit(Callable task)

异步执行任务，返回Future对象，并且接收的是callable，有返回值，可以通过future.get获取。future.get方法会阻塞线程。

结果为:

submit(Runnable task, T result)

返回Future对象。但是接收的是Runnable对象，是没有返回值的，第二个参数就规定了一个返回值，然后通过future.get获取的就是第二个参数。

结果为:

submit(Runnable task)

返回Future对象。但是接收的是Runnable对象，是没有返回值的。通过future.get获取的是null。

结果为:

invokeAll(Collection<? extends Callable> tasks)

多个任务一起执行，返回所有任务执行完成的集合，返回结果是没有先后顺序的。

结果为:

invokeAll(Collection<? extends Callable> tasks,long timeout, TimeUnit unit)

结果为:

同时执行所有任务，限制超时时间。在规定时间内没有完成，会报CancellationException异常，表示此方法超时。注意创建线程池的方式，如果是多个线程，
则会统一分配。如果只是一个线程，那所有任务都是一个线程执行的。一个线程超时的话，那所有任务都显示超时。

invokeAny(Collection<? extends Callable> tasks)

多个任务一起执行，返回运行最快的值。只返回一个值。

结果为:

构造方法

ExecutorService executorService = Executors.newFixedThreadPool(5)

创建有规定个数线程的线程池，超出的线程会在队列中等待。返回的是ThreadPoolExecutor。

ExecutorService executorService = Executors.newSingleThreadExecutor()

创建唯一线程的线程，保证所有任务都按照指定顺序执行。返回的是ThreadPoolExecutor。

ExecutorService executorService = Executors.newCachedThreadPool()

创建一个可以缓存的线程池，如果线程池不够则主动扩充线程的个数。返回的是ThreadPoolExecutor。

ExecutorService executorService = Executors.newScheduledThreadPool(5)

创建规定个数线程的线程池，支持定时以及周期性任务执行。返回的是ScheduledThreadPoolExecutor。

ExecutorService executorService = Executors.newSingleThreadScheduledExecutor()

创建唯一线程的线程，支持定时以及周期性任务执行。返回的是ScheduledThreadPoolExecutor。
